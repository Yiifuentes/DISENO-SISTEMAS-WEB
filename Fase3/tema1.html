Etiquetas semanticas, css

llamadas hojas de estilo en cascada, CSS o Cascading Style Sheets abren un nuevo abanico de posibilidades para los creadores de páginas web. La idea que se encuentra detrás del desarrollo de CSS es separar la estructura de un documento de su presentación o aspecto, como por ejemplo, la combinación de XHTML y CSS.

Para qué sirven CSS

Las hojas de estilo constituyen el complemento ideal para HTML o XHTML. Su misión es definir la apariencia y el estilo de sus elementos. Por ejemplo, podemos especificar en una línea que todo el texto enfatizado del documento use la fuente Verdana en color verde con un tamaño de 12 puntos.

Tipos de hojas de estilo

Las hojas de estilo, según la manera de aplicarse, se pueden dividir en dos grupos: Hoja de estilo interna o residente: Sólo afecta al documento en el que se crea. se Se define con la etiqueta, STYLE que se inserta en la cabecera del documento, entre las etiquetas ‹HEAD> y .

Hoja de estilo externa: Puede afectar a cualquier documento que se vincule con ella. Contiene toda la información en un documento independiente del documento HTML. Se almacena con extensión .CSS. Las hojas de estilo, según al contenido al que afectan, se pueden dividir en tres tipos: De clase: Afecta al bloque de texto sobre el que se aplique. Por tanto puede modificar cualquier contenido determinado dentro de un documento, : no a los demás párrafos del mismo. simplemente con aplicar la hoja a ese contenido concreto. Por ejemplo, se puede aplicar un estilo a un a párrafo todo el documento. De etiqueta: Se aplica a una etiqueta determinada y afectará a cualquier contenido definido a por la etiqueta vinculada. Por ejemplo, se puede aplicar un estilo a todos los textos que estén De identificador Id: Mediate un identificador se pueden marcar los textos a los que afectará el estilo.

Etiquetas semanticas estructurales

Las etiquetas semánticas nos permiten definir la estructura de las paginas web en HTML, y nos sirven para que los motores de búsqueda, sepa con exactitud qué partes de su contenido correspondan a cada una de las partes típicas de un sitio.
